# 予約アプリ 要件定義書（Day5）

## 1. 目的

大病院における診療予約を Web（スマートフォン）から簡単・迷わず行えるようにする。
電話予約や窓口対応の負担を減らし、患者・病院双方の利便性を向上させる。

---

## 2. 想定ユーザー

- スマートフォンを日常的に利用する患者
- 年齢層：20代〜高齢者まで幅広い
- ITリテラシーは高くないユーザーも含む  
  → **直感的で分かりやすいUIが必須**

---

## 3. 利用環境

| 項目 | 内容 |
|------|------|
| デバイス | スマートフォン（優先）、PC（補助） |
| ブラウザ | Chrome / Safari 想定 |
| アプリ形式 | Webアプリ（インストール不要） |

---

## 4. 機能一覧

### 【必須機能】

#### 4-1. ユーザー登録・ログイン

- メールアドレス + パスワードによる登録
- ログイン / ログアウト
- **Firebase Authentication** を使用
- ログイン状態を保持（リロード後も有効）

#### 4-2. 日付選択カレンダー

- 月表示のカレンダー
- 予約可能日のみ選択可能
- 過去日は選択不可
- 選択した日付に応じて予約枠を切り替え表示

#### 4-3. 予約登録

以下の手順で予約を行う：

1. **大分類を選択**  
   内科系 / 外科系 / 小児・女性 / 検査 / リハビリ
2. **診療科を選択**  
   例：循環器内科、整形外科、眼科 など
3. **予約目的を選択**  
   初診 / 再診 / 検査 / 処置 / 相談 / 健診
4. **担当医を選択**  
   医師ごとの勤務時間を表示
5. **30分刻みで予約枠を表示**  
   予約時間を選択し、予約を確定

- 予約情報は **Firebase Firestore** に保存
- 同一時間帯の重複予約は不可

---

## 5. 画面一覧

| 画面 | 主な要素 |
|------|----------|
| ログイン画面 | メールアドレス入力、パスワード入力、新規登録リンク |
| 新規登録画面 | メールアドレス、パスワード、登録ボタン |
| カレンダー選択画面 | 月表示カレンダー、日付選択 |
| 予約入力フォーム画面 | 大分類・診療科・予約目的・担当医・時間枠選択 |
| 予約完了画面 | 予約内容の確認表示、「予約が完了しました」のメッセージ |

---

## 6. 非機能要件

- 操作は **3〜5タップ以内**で予約完了できること
- スマホ画面で文字が小さくなりすぎない
- レスポンスは体感で遅く感じないこと
- エラー時は分かりやすいメッセージを表示
